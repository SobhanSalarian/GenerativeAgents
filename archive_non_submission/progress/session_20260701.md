## 1. Comment #4 gap closed — neutral labels + option order randomisation

### What was missing

`run_ic_counterbalance.py` (previous session) only rotated WHICH option
was correct (A/B/C). It did not control for:
- Alphabetical label bias — LLMs may prefer "A" because it is first alphabetically
- Positional bias — the correct option always appeared in the same display
  position (A always 1st, B always 2nd, C always 3rd)
- Semantic priming — option descriptions ("creative studio") may be more
  salient than others

### New files created (original `information_consensus.py` NOT modified)

**`scenarios/information_consensus_neutral.py`**

Subclass of `InformationConsensus`. Key design decisions:
- Labels: A→X, B→Y, C→Z throughout all agent-visible text
- Descriptions: "Community development proposal X/Y/Z" (no semantic content)
- Display order: Latin square rotation over all 6 permutations of [X,Y,Z].
  `_consensus_context(step_num)` sets `self._step_display_order` (used by
  `_ask_agent` for the vote instruction in the same step). Over 30 steps,
  each label appears in each position exactly 10 times.
- Internal tracking: always A/B/C (pipeline-compatible)
- `_init_kwargs()` returns `{true_option, signal_counts}` for `clone_scenario()`
- Overrides: `_consensus_context`, `_ask_agent`, `_record_scenario_memories`,
  `_generate_llm_reflection`, `_check_signal_disclosed`
- NOT overridden (inherited correctly): `step()`, `setup()`, `is_complete()`,
  `_deterministic_reflection` (generic text, no label letters)

**`run_ic_neutral_labels.py`**

3 rotations × baseline+full × 5 trials × 30 steps = 30 trials total.

```
correct_A_as_X → experiment_results_ic_neutral_correct_A_as_X/
correct_B_as_Y → experiment_results_ic_neutral_correct_B_as_Y/
correct_C_as_Z → experiment_results_ic_neutral_correct_C_as_Z/
```

### Bugs found and fixed during review

1. `vote_options` formatting: initial version used string join that produced
   `"X", "Y", "Z"` (missing "or"). Fixed to `"X", "Y", or "Z"`.
2. `_to_internal` substring scan: single-char scan had false-positive risk.
   Replaced with explicit priority checks: exact → last word → first char →
   bare internal key fallback.
3. Missing option order randomisation: display order was fixed (sorted A/B/C
   → X/Y/Z). Fixed with Latin square rotation over 6 permutations.
