## Critical assessment of the reframing

**Strengths:**
- The believability–validity gap frame is genuinely stronger and more publishable than the original confirmatory framing
- The IC cascade result (believable agents converging on the wrong answer) is a clean, counterintuitive hook
- Methodological caveats are well-handled throughout — planning confound, IC binary evidence, within-study threshold, single architecture

**Risks and fragilities:**
1. **H6 (focal-point norm) depends on P1.** The full condition uses deterministic rule-based reflection, so 2 unique strings may be an implementation artifact, not a Schelling focal-point phenomenon. Until P1 runs (or the full_llm_reflection rerun completes), H6 is plausible but not confirmed. The full_llm_reflection rerun (15/20 trials still outstanding) is directly relevant.
2. **H7 will likely be underpowered.** With 20 trials per condition in CD and the within-condition test as the credibility bar, the classifier has very limited N. Exploratory framing is necessary.
3. **IC binary evidence limits C2.** The gap demonstration rests on the IC baseline cascade, but the 3 enriched IC conditions have near-zero variance — effectively one data point per condition. The signed discrepancy metric will be most informative in CD data.
4. **"Necessary" is too strong.** The abstract uses "necessary but not sufficient" for the believability–coordination relationship. The data shows a lower boundary, not formal necessity. Recommend: "consistent with necessity within this study" or "exhibiting an observed lower boundary."

## Implementation plan

### Phase 1 — Audit the data (no code)

Verify the reframing's empirical claims against actual output files before writing any code:
- Check full condition logs → confirm reflection saturation = 2 unique strings
- Check IC baseline trials → confirm high-believability agents converge on wrong option

### Phase 2 — H5/C2: Signed believability–validity discrepancy metric

**File:** `experiment_analysis.py`

Add `compute_discrepancy_metric()`:
```
discrepancy = composite_believability − coordination_validity_score
```
- Per trial; coordination validity is a normalised macro score (not just binary)
- Positive discrepancy = believable agents, poor coordination (the gap case)
- Flag large-positive cases → become the C2 demonstration

### Phase 3 — H6/C3-ii: Reflection saturation analysis

**File:** `experiment_analysis.py`

Add `analyze_reflection_saturation()`:
- Read reflection strings from full condition logs across all trials and steps
- Count unique strings over time (diversity curve: unique count vs. step)
- Report when saturation occurs and what the 2 strings are
- Output: figure + count → direct thesis evidence for focal-point interpretation

### Phase 4 — H7/C4: Leakage-safe early-warning classifier + failure taxonomy

**New file:** `early_warning.py`

**Early warning (H7):**
- Features: micro scores (BC, MC, PP, RN) from first K steps per trial (K sweep: 5, 10, 15)
- Label: coordination collapse (binary)
- Evaluation: trial-level leave-one-out CV — never split within a trial
- Explicit within-condition test: run classifier within each condition separately
- Output: precision, recall, AUC, lead time (earliest reliable K)

**Failure taxonomy (C4):**
- For collapsed trials: cluster micro-failure patterns (which sub-dimension breaks first, sequence, speed)
- Produce 3–5 recurring failure types linking micro breakdown to macro collapse
- Each type: label + representative trace

### Phase 5 — P1 experiment design

**Files:** `experiment_conditions.py`, `experiment_runner.py`

Add two new conditions:
- `static_reflection_injection` — memory_planning + one of the two saturated strings injected at each step
- `static_reflection_placebo` — memory_planning + tone/length-matched string with no coordination content

Whether to *run* P1 depends on API budget and HREC/timeline decisions.

### Phase 6 — Wire into analysis entrypoint

Update `experiment_analysis.py` main function to run H1–H7 in sequence, unified report. Hypothesis labels updated from H1–H4 to H1–H8 (H7 flagged exploratory, H8 planned).
